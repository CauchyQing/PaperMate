# PaperMate - 交互式论文阅读助手 需求文档

## 1. 项目概述

### 1.1 产品定位
一款基于 Electron 的桌面端学术论文阅读工具，集成 AI 助手功能，帮助研究人员高效阅读、理解和管理学术论文。

### 1.2 核心目标
- 提供沉浸式的论文阅读体验
- 通过 AI 助手实时解答论文中的疑问
- 建立项目化的论文知识库，支持多维度分类检索

### 1.3 工作区模式（核心设计理念）
PaperMate 采用**项目/工作区（Workspace）**模式，类似于 VS Code / Cursor / Codex：

- 用户**选择一个本地文件夹**作为工作区打开
- 所有论文分类、阅读进度、对话历史**保存在该文件夹内**
- 便于项目协作（整个文件夹可共享、版本控制）
- 支持同时打开多个工作区（多窗口）

## 3. 工作区系统设计（核心）

### 3.1 工作区概念

**工作区（Workspace）** 是一个包含 `.papermate/` 隐藏目录的文件夹，代表一个独立的研究项目。

| 特性 | 说明 |
|------|------|
| **独立性** | 每个工作区有独立的数据库、设置、历史记录 |
| **便携性** | 整个文件夹可复制、压缩、版本控制（Git） |
| **多开支持** | 可同时打开多个工作区（多窗口） |
| **路径无关** | 论文使用相对路径存储，移动文件夹不影响 |

### 3.2 工作区文件结构

```
workspace-folder/                   # 用户项目文件夹
├── .papermate/                     # PaperMate 工作区目录
│   ├── database/
│   │   └── papermate.db            # SQLite 数据库
│   ├── settings.json               # 工作区配置
│   ├── state.json                  # 运行状态（窗口大小、打开的文件等）
│   └── cache/
│       ├── thumbnails/             # PDF 缩略图缓存
│       └── snapshots/              # 截图缓存
└── [用户文件...]                   # 论文、笔记等
```

### 3.3 工作区配置（settings.json）

```typescript
interface WorkspaceSettings {
  // 版本号
  version: '1.0.0';
  
  // 工作区信息
  name: string;                    // 工作区名称（默认文件夹名）
  description?: string;            // 描述
  createdAt: number;               // 创建时间
  
  // 论文存储配置
  papersRoot: './papers';          // 论文根目录（相对路径）
  
  // 分类配置
  categories: {
    enabled: string[];             // 启用的分类维度
    custom: Array<{
      id: string;
      name: string;
      type: 'single' | 'multiple';
      options?: string[];          // 预设选项
    }>;
  };
  
  // AI 助手配置
  agent: {
    systemPrompt?: string;         // 自定义系统提示词
    model?: string;                // 默认模型
    contextLength?: number;        // 上下文长度限制
  };
  
  // 界面设置
  ui: {
    sidebarWidth: number;
    pdfSidebarVisible: boolean;
    theme: 'light' | 'dark' | 'system';
  };
}
```

### 3.5 数据库路径相对化

为适应工作区模式，数据库中的文件路径**全部使用相对路径**（相对于工作区根目录）：

```typescript
// 论文表中的路径存储
{
  id: 'paper-001',
  // ❌ 不要这样
  filePath: '/Users/xxx/research/papers/attention.pdf',
  // ✅ 应该这样
  filePath: './papers/attention.pdf',
  
  // 其他元数据...
}
```

**解析逻辑**：
```typescript
function resolveWorkspacePath(relativePath: string, workspaceRoot: string): string {
  return path.resolve(workspaceRoot, relativePath);
}

function toRelativePath(absolutePath: string, workspaceRoot: string): string {
  return './' + path.relative(workspaceRoot, absolutePath);
}
```

## 5. 功能模块详细需求

### 5.1 工作区管理模块

#### 5.1.1 打开工作区
- 通过欢迎页"打开文件夹"按钮
- 通过菜单 `文件 → 打开文件夹` (Ctrl+O / Cmd+O)
- 拖拽文件夹到应用窗口

**流程**：
1. 用户选择文件夹
2. 检查是否存在 `.papermate/settings.json`
   - **存在**：直接加载工作区
   - **不存在**：询问"是否将此文件夹初始化为 PaperMate 工作区？"
3. 初始化时创建 `.papermate/` 目录结构
4. 加载工作区，显示主界面

#### 5.1.2 创建工作区
- **方式1**：欢迎页"新建工作区"
- **方式2**：菜单 `文件 → 新建工作区`

**流程**：
1. 选择空文件夹或创建新文件夹
2. 填写工作区名称和描述（可选）
3. 选择论文存储位置（默认 `./papers`）
4. 初始化 `.papermate/` 结构
5. 打开工作区

#### 5.1.3 关闭/切换工作区
- 关闭当前工作区：返回欢迎页
- 切换工作区：关闭当前，打开新的（单窗口模式）
- 新窗口打开：保持当前，新开窗口（多窗口模式）

#### 5.1.4 工作区设置
设置项：
- 工作区名称、描述
- 论文根目录（可更改）
- 启用的分类维度
- 自定义标签
- AI 助手配置

### 5.2 PDF 阅读器模块

#### 5.2.1 基础阅读功能
- **多标签页**: 顶部标签栏，支持同时打开多篇论文
  - 显示论文标题（缩略）
  - 支持关闭标签、重新排序
  - 未保存的阅读进度提示
  
- **PDF 渲染**: 
  - 使用 react-pdf 渲染 PDF
  - 支持缩放（50% - 300%）
  - 支持页面导航（上一页/下一页/跳转到指定页）
  - 支持缩略图侧边栏（可选折叠）
  - 暗黑/亮色模式适配

#### 5.2.2 框选功能（关键设计）
**方案选择: 混合模式（推荐）**

| 场景 | 方式 | 说明 |
|------|------|------|
| 文本内容 | 文本提取 | 用户划选文本，直接提取文字，精度高、token 少 |
| 图表/公式 | 截图 | 框选区域生成图片，AI 进行视觉理解 |

- **文本划选**: 
  - 鼠标拖动划选 PDF 文本
  - 弹出浮动工具栏（解释/翻译/引用/收藏）
  - 选中文本自动高亮（临时/永久）

- **区域截图**:
  - 快捷键 `Ctrl+Shift+S` 或工具栏按钮进入截图模式
  - 鼠标拖动框选任意区域
  - 生成 base64 图片发送到 AI

- **浮动工具栏按钮**:
  - 🔍 解释 (Explain)
  - 🌐 翻译 (Translate)

### 5.3 AI 助手模块

#### 5.3.1 对话界面（左侧栏）
- **布局**: 左侧边栏，宽度可调（最小 350px，最大 600px）
- **消息展示**:
  - 用户消息（选中的文本/图片）
  - AI 回复（Markdown 渲染，支持代码块、公式）
  - LaTeX 公式渲染 (KaTeX)
  - 消息时间戳
  
- **输入方式**:
  - 文本输入框（支持 Markdown）
  - 快捷按钮（继续/简化/详细解释/举例说明）
  - 支持粘贴图片

#### 5.3.2 Agent SDK 集成
```typescript
// 核心配置
const agentOptions: ClaudeAgentOptions = {
  allowedTools: ['Read', 'WebSearch', 'WebFetch'],
  permissionMode: 'acceptEdits',
  systemPrompt: `你是 PaperMate，一个专业的学术助手。你可以：
1. 解释学术论文中的复杂概念
2. 翻译外文文献（保持学术术语准确）
3. 分析图表和公式
4. 搜索相关文献补充背景知识
5. 帮助总结论文要点

回复原则：
- 解释要深入浅出，适合研究者理解
- 数学公式使用 LaTeX 格式
- 引用外部资料时注明来源`,
};
```

#### 5.3.3 上下文压缩策略（重要）
由于论文对话可能很长，需要智能管理上下文：

**策略 1: 会话分段**
- 每次对话创建独立的 "话题" (Topic)
- 用户可以在历史栏切换话题
- 每个话题独立维护上下文

**策略 2: 摘要替换（可选高级功能）**
- 当对话超过 10 轮或 token 阈值时
- 让 AI 生成对话摘要，替换早期消息
- 保留关键结论，丢弃中间推理

**策略 3: 引用原文**
- 用户消息引用论文具体内容时，只保留引用 + 用户问题
- 不保留 AI 的中间推理过程

**策略 4: 显式重置**
- 提供 "新话题" 按钮，清空上下文重新开始
- 提示用户当前论文主题，避免混淆

```sql
-- 标签表（主题、关键词等）
CREATE TABLE tags (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    type TEXT NOT NULL,               -- 'topic', 'keyword', 'custom'
    color TEXT,                       -- 标签颜色 #RRGGBB
    createdAt INTEGER NOT NULL
);

-- 论文-标签关联表
CREATE TABLE paper_tags (
    paperId TEXT NOT NULL,
    tagId TEXT NOT NULL,
    PRIMARY KEY (paperId, tagId),
    FOREIGN KEY (paperId) REFERENCES papers(id) ON DELETE CASCADE,
    FOREIGN KEY (tagId) REFERENCES tags(id) ON DELETE CASCADE
);

-- 收藏/高亮内容
CREATE TABLE highlights (
    id TEXT PRIMARY KEY,
    paperId TEXT NOT NULL,
    pageNumber INTEGER NOT NULL,
    selectedText TEXT,
    note TEXT,                        -- 用户注释
    color TEXT DEFAULT '#FFD700',     -- 高亮颜色
    rect TEXT,                        -- 矩形区域（JSON: {x, y, width, height}）
    createdAt INTEGER NOT NULL,
    FOREIGN KEY (paperId) REFERENCES papers(id) ON DELETE CASCADE
);

-- 对话历史表
CREATE TABLE conversations (
    id TEXT PRIMARY KEY,
    paperId TEXT,
    topic TEXT NOT NULL,              -- 对话主题
    createdAt INTEGER NOT NULL,
    updatedAt INTEGER NOT NULL,
    messageCount INTEGER DEFAULT 0,
    contextSummary TEXT,              -- 上下文摘要（用于压缩）
    FOREIGN KEY (paperId) REFERENCES papers(id) ON DELETE SET NULL
);

CREATE TABLE messages (
    id TEXT PRIMARY KEY,
    conversationId TEXT NOT NULL,
    role TEXT NOT NULL,               -- 'user' | 'assistant'
    content TEXT NOT NULL,
    contentType TEXT DEFAULT 'text',  -- 'text' | 'image' | 'mixed'
    metadata TEXT,                    -- JSON: {pageNumber, selectedText, etc.}
    timestamp INTEGER NOT NULL,
    FOREIGN KEY (conversationId) REFERENCES conversations(id) ON DELETE CASCADE
);
```

#### 5.5.2 分类维度详解

| 维度 | 类型 | 说明 | 来源 |
|------|------|------|------|
| **年份** | 时间 | 发表年份，支持范围筛选 | 自动提取 / 手动填写 |
| **期刊/会议** | 文本 | CVPR、ICML、Nature 等 | 自动提取 / 手动填写 |
| **主题** | 标签 | 大类别：CV、NLP、RL 等 | 用户定义 + AI 辅助建议 |
| **关键词** | 标签 | 具体技术：Transformer、GNN 等 | 自动提取 + 用户编辑 |
| **阅读状态** | 枚举 | 未读/阅读中/已读/精读 | 系统计算 |
| **重要性** | 枚举 | ⭐ - ⭐⭐⭐⭐⭐ | 用户标记 |

## 7. 关键交互流程

### 7.1 打开论文并提问

```
1. 用户点击"导入论文"或拖入 PDF
   ↓
2. 系统解析 PDF 元数据（标题、作者等）存储到 SQLite
   ↓
3. 在右侧打开 PDF，左侧自动创建新会话
   ↓
4. 用户划选文本 → 弹出工具栏 → 点击"解释"
   ↓
5. 选中文本发送到 Agent SDK
   ↓
6. AI 回复展示在左侧对话区
   ↓
7. 对话自动保存到数据库
```

### 7.2 多维度分类流程

```
1. 用户进入"图书馆"页面
   ↓
2. 系统展示所有论文
   ↓
3. 用户可以通过以下方式分类：
   a. 右键论文 → 添加标签
   b. 批量选择 → 批量打标签
   c. 使用筛选栏组合筛选
   ↓
4. 点击标签可快速筛选相关论文
```

## 8. 数据存储策略

### 8.1 工作区文件存储

每个工作区的文件存储在 `.papermate/` 目录下：

```
workspace-folder/
├── .papermate/
│   ├── database/
│   │   └── papermate.db            # SQLite 数据库
│   ├── settings.json               # 工作区设置
│   ├── state.json                  # 运行状态
│   └── cache/                      # 缓存
│       ├── thumbnails/             # PDF 缩略图
│       └── snapshots/              # 截图缓存
└── [用户论文文件...]
```

### 8.2 全局应用数据

存储在用户目录 `~/.papermate/`：

- **全局配置**: `~/.papermate/global-config.json`（最近工作区列表、API key）
- **日志**: `~/.papermate/logs/`
- **临时文件**: `~/.papermate/temp/`

### 8.3 备份策略
- 支持手动导出数据库
- 支持自动备份（可选）

## 10. 技术难点与解决方案

### 10.1 PDF 文本提取精度
- **方案**: 使用 pdfjs-dist + 自定义文本层
- **备选**: 对扫描版 PDF 使用 OCR (Tesseract.js)

### 10.2 长对话上下文管理
- **方案**: 按话题分会话 + 用户主动管理
- **优化**: 提供 "总结对话" 按钮生成摘要

### 10.3 大量论文性能
- **方案**: 数据库索引 + 虚拟滚动
- **分页**: 图书馆页面分页加载

### 10.4 Agent SDK 错误处理
- **方案**: 包装调用，提供重试机制
- **用户提示**: 网络错误时显示友好提示
