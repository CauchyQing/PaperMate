# PaperMate - 交互式论文阅读助手 需求文档

## 1. 项目概述

### 1.1 产品定位

一款基于 Electron 的桌面端学术论文阅读工具，集成 AI 助手功能，帮助研究人员高效阅读、理解和管理学术论文。

### 1.2 核心目标

- 提供沉浸式的论文阅读体验
- 通过 AI 助手实时解答论文中的疑问
- 建立项目化的论文知识库，支持多维度分类检索

### 1.3 工作区模式（核心设计理念）

PaperMate 采用\*\*项目/工作区（Workspace）\*\*模式，类似于 VS Code / Cursor / Codex：

- 用户**选择一个本地文件夹**作为工作区打开
- 所有论文分类、阅读进度、对话历史**保存在该文件夹内**
- 便于项目协作（整个文件夹可共享、版本控制）
- 支持同时打开多个工作区（多窗口）

## 3. 工作区系统设计（核心）

### 3.1 工作区概念

**工作区（Workspace）** 是一个包含 `.papermate/` 隐藏目录的文件夹，代表一个独立的研究项目。

| 特性       | 说明                    |
| -------- | --------------------- |
| **独立性**  | 每个工作区有独立的数据库、设置、历史记录  |
| **便携性**  | 整个文件夹可复制、压缩、版本控制（Git） |
| **多开支持** | 可同时打开多个工作区（多窗口）       |
| **路径无关** | 论文使用相对路径存储，移动文件夹不影响   |

### 3.2 工作区文件结构

```
workspace-folder/                   # 用户项目文件夹
├── .papermate/                     # PaperMate 工作区目录
│   ├── database/
│   │   └── papermate.db            # SQLite 数据库
│   ├── settings.json               # 工作区配置
│   ├── state.json                  # 运行状态（窗口大小、打开的文件等）
│   └── cache/
│       ├── thumbnails/             # PDF 缩略图缓存
│       └── snapshots/              # 截图缓存
└── [用户文件...]                   # 论文、笔记等
```

### 3.3 工作区配置（settings.json）

```typescript
interface WorkspaceSettings {
  // 版本号
  version: '1.0.0';
  
  // 工作区信息
  name: string;                    // 工作区名称（默认文件夹名）
  description?: string;            // 描述
  createdAt: number;               // 创建时间
  
  // 论文存储配置
  papersRoot: './papers';          // 论文根目录（相对路径）
  
  // 分类配置
  categories: {
    enabled: string[];             // 启用的分类维度
    custom: Array<{
      id: string;
      name: string;
      type: 'single' | 'multiple';
      options?: string[];          // 预设选项
    }>;
  };
  
  // AI 助手配置
  agent: {
    systemPrompt?: string;         // 自定义系统提示词
    model?: string;                // 当前使用的模型
    providerId?: string;           // 当前使用的供应商 ID
    translateLang?: string;        // 默认翻译目标语言（默认 'zh-CN'）
    contextWindowSize?: number;    // 滑动窗口大小（保留最近 N 轮对话）
    autoAnalyze?: boolean;         // 导入论文时是否自动 AI 分析
  };
  
  // 界面设置
  ui: {
    sidebarWidth: number;
    pdfSidebarVisible: boolean;
    theme: 'light' | 'dark' | 'system';
  };
}
```

### 3.5 数据库路径相对化

为适应工作区模式，数据库中的文件路径**全部使用相对路径**（相对于工作区根目录）：

```typescript
// 论文表中的路径存储
{
  id: 'paper-001',
  // ❌ 不要这样
  filePath: '/Users/xxx/research/papers/attention.pdf',
  // ✅ 应该这样
  filePath: './papers/attention.pdf',
  
  // 其他元数据...
}
```

**解析逻辑**：

```typescript
function resolveWorkspacePath(relativePath: string, workspaceRoot: string): string {
  return path.resolve(workspaceRoot, relativePath);
}

function toRelativePath(absolutePath: string, workspaceRoot: string): string {
  return './' + path.relative(workspaceRoot, absolutePath);
}
```

## 5. 功能模块详细需求

### 5.1 工作区管理模块

#### 5.1.1 打开工作区

- 通过欢迎页"打开文件夹"按钮
- 通过菜单 `文件 → 打开文件夹` (Ctrl+O / Cmd+O)
- 拖拽文件夹到应用窗口

**流程**：

1. 用户选择文件夹
2. 检查是否存在 `.papermate/settings.json`
   - **存在**：直接加载工作区
   - **不存在**：询问"是否将此文件夹初始化为 PaperMate 工作区？"
3. 初始化时创建 `.papermate/` 目录结构
4. 加载工作区，显示主界面

#### 5.1.2 创建工作区

- **方式1**：欢迎页"新建工作区"
- **方式2**：菜单 `文件 → 新建工作区`

**流程**：

1. 选择空文件夹或创建新文件夹
2. 填写工作区名称和描述（可选）
3. 选择论文存储位置（默认 `./papers`）
4. 初始化 `.papermate/` 结构
5. 打开工作区

#### 5.1.3 关闭/切换工作区

- 关闭当前工作区：返回欢迎页
- 切换工作区：关闭当前，打开新的（单窗口模式）
- 新窗口打开：保持当前，新开窗口（多窗口模式）

#### 5.1.4 工作区设置

设置项：

- 工作区名称、描述
- 论文根目录（可更改）
- 启用的分类维度
- 自定义标签

### 5.3 AI 助手模块

#### 5.3.1 技术方案：OpenAI 兼容接口

**为什么不用 Claude Agent SDK？**

- Claude Agent SDK 仅支持 Anthropic 模型，无法连接其他平台（如 OpenAI、DeepSeek、Qwen、本地部署模型等）
- 用户可能需要使用不同的模型，或者公司/学校有指定的 API 服务

**方案：统一使用 OpenAI Chat Completions 兼容格式**

- 大多数大模型平台都提供 OpenAI 兼容接口（Anthropic、OpenAI、DeepSeek、Qwen、Ollama 本地模型等）
- 用户只需配置 API Base URL + API Key + 模型名称即可接入任意平台
- 也可以调用本地已部署的 Claude Code / Ollama 等服务

```typescript
// AI 服务配置（存储在全局配置中）
interface AIProviderConfig {
  id: string;                    // 供应商 ID
  name: string;                  // 显示名称（如 "OpenAI", "DeepSeek", "本地模型"）
  baseUrl: string;               // API Base URL
  apiKey: string;                // API Key（加密存储）
  models: string[];              // 可用模型列表
  defaultModel: string;          // 默认模型
  maxTokens?: number;            // 最大生成 token 数
  supportsVision?: boolean;      // 是否支持图片输入（用于截图提问）

// 预设供应商模板（用户可直接选择）
```

#### 5.3.2 核心 AI 调用服务

在 Electron **主进程**中实现 AI 调用，通过 IPC 与渲染进程通信：

```typescript
// src/main/services/ai-service.ts
class AIService {
  private config: AIProviderConfig;

  // 基础对话（流式输出）
  async chat(messages: ChatMessage[], options?: ChatOptions): AsyncGenerator<string>;

  // 翻译（专用提示词）
  async translate(text: string, targetLang: string, context?: string): AsyncGenerator<string>;

  // 论文分析（读取 PDF 内容，智能打标签）
  async analyzePaper(pdfText: string): Promise<PaperAnalysis>;

  // 解释选中内容
  async explain(selectedText: string, paperContext?: string): AsyncGenerator<string>;

  // 图片理解（截图提问）
  async analyzeImage(imageBase64: string, question: string): AsyncGenerator<string>;
}

interface ChatMessage {
  role: 'system' | 'user' | 'assistant';
  content: string | ContentPart[];  // 支持多模态（文本 + 图片）
}

interface ContentPart {
  type: 'text' | 'image_url';
  text?: string;
  image_url?: { url: string };  // base64 data URL
}

interface PaperAnalysis {
  suggestedTitle: string;
  suggestedJournal?: string;
  suggestedYear?: number;
  suggestedTopics: string[];     // 建议的主题标签
  suggestedKeywords: string[];   // 建议的关键词标签
  summary: string;               // 论文摘要
}
```

**对话栏头部**:

- 当前会话标题（可编辑）
- 关联论文名称（点击跳转）
- "新建会话"按钮

#### 5.3.4 功能一：论文翻译

**核心场景**: 用户阅读英文论文时，需要逐段翻译

**交互方式**:

1. **划选翻译**: 在 PDF 上划选文字 → 浮动工具栏点击"翻译" → 翻译结果显示在对话区
2. **截图翻译**: 框选包含公式/图表的区域 → 点击"翻译" → AI 识别并翻译内容
3. **连续翻译模式**:
   - 用户可以一段一段地选取文字发送翻译
   - 所有翻译结果在同一个会话中连续展示
   - AI 会自动理解上文语境，保持翻译连贯性（比如代词指代、术语统一）

**翻译专用系统提示词**:

```
你是一位专业的学术论文翻译助手。请将以下学术论文内容翻译为中文。

翻译原则：
1. 保持学术术语的准确性，对于专业术语在首次出现时标注英文原文
2. 数学公式保持 LaTeX 格式不翻译，仅翻译其周围的自然语言
3. 保持原文的段落结构和逻辑层次
4. 对于缩写词（如 CNN、LSTM）保留原文，在首次出现时给出中文全称
5. 保持与之前翻译内容的术语一致性
```

**上下文维护策略（翻译场景）**:

- 翻译是 token 密集型任务，一篇 20 页的论文完整翻译可能消耗 50K+ token
- 解决方案：
  1. 每段翻译结果保存到会话中，但发送 API 时只携带最近 N 段的上下文（滑动窗口）
  2. 维护一个"术语表"：AI 在翻译过程中自动提取关键术语映射（如 "attention mechanism" → "注意力机制"），术语表始终随请求发送
  3. 用户可导出完整翻译结果为 Markdown 文件

#### 5.3.5 功能二：上下文管理系统

**核心挑战**: 论文讨论和翻译都涉及大量 token，需要智能管理上下文

**上下文策略（分层设计）**:

```
┌─────────────────────────────────────────────┐
│ 系统提示词（始终存在）                         │  ← 固定 token
│ - 角色设定                                   │
│ - 当前论文元信息（标题、作者、摘要）            │
│ - 术语表（翻译场景专用）                       │
├─────────────────────────────────────────────┤
│ 上下文摘要（当对话超过阈值时生成）              │  ← 压缩 token
│ - 之前讨论的要点总结                           │
│ - 用户关注的核心问题                           │
├─────────────────────────────────────────────┤
│ 最近 N 轮对话（滑动窗口）                      │  ← 活跃 token
│ - 保留最近的完整对话                           │
│ - 确保 AI 理解当前语境                         │
├─────────────────────────────────────────────┤
│ 当前用户输入                                   │  ← 新增 token
└─────────────────────────────────────────────┘
```

**触发机制**:

- 当历史消息总 token 数超过模型上下文窗口的 60% 时，自动触发摘要压缩
- 摘要由 AI 生成，用户可查看和编辑
- 用户也可手动点击"总结对话"按钮

**token 估算**:

```typescript
// 简单的 token 估算（中文约 1.5 token/字，英文约 1 token/word）
function estimateTokens(text: string): number {
  const chineseChars = (text.match(/[\u4e00-\u9fff]/g) || []).length;
  const englishWords = text.replace(/[\u4e00-\u9fff]/g, '').split(/\s+/).filter(Boolean).length;
  return Math.ceil(chineseChars * 1.5 + englishWords * 1.3);
}
```

#### 5.3.6 功能三：智能论文分析与标签分配

**场景**: 导入新论文时，AI 自动分析论文内容并建议分类标签

**工作流程**:

1. 用户导入 PDF 论文
2. 系统提取 PDF 文本内容（前 3 页，包含标题、摘要、引言）
3. 发送到 AI 进行分析
4. AI 返回结构化结果：
   - 建议标题（如果文件名不规范）
   - 发表年份、期刊/会议
   - 建议主题标签（从已有标签中选择 + 建议新标签）
   - 建议关键词标签
   - 论文简要摘要
5. 用户在确认弹窗中审核和修改 AI 的建议
6. 确认后自动应用到论文分类

**分析请求格式**:

```typescript
// 发送给 AI 的分析请求
const analyzePrompt = `
请分析以下学术论文内容，返回 JSON 格式的分析结果。

已有的主题标签：${existingTopics.join(', ')}
已有的关键词标签：${existingKeywords.join(', ')}

论文内容（前3页）：
${paperText}

请返回以下 JSON 格式：
{
  "title": "论文标题",
  "year": 2024,
  "journal": "期刊或会议名称",
  "topics": ["从已有标签中选择，或建议新主题"],
  "keywords": ["从已有标签中选择，或建议新关键词"],
  "summary": "100字以内的论文摘要"
}
`;
```

**批量分析**: 支持选择多篇论文一键分析，排队处理并显示进度

### 5.5 多维度分类系统（核心功能）

#### 5.5.1 数据库设计（SQLite）

```sql
-- 论文主表
CREATE TABLE papers (
    id TEXT PRIMARY KEY,
    filePath TEXT NOT NULL,           -- 本地文件路径
    fileName TEXT NOT NULL,
    fileSize INTEGER,
    title TEXT,                       -- 论文标题（可自动提取/手动编辑）
    authors TEXT,                     -- 作者（JSON 数组）
    abstract TEXT,
    doi TEXT,
    url TEXT,
    -- 分类字段
    publishYear INTEGER,              -- 发表年份
    journal TEXT,                     -- 期刊/会议名称
    -- 元数据
    importedAt INTEGER NOT NULL,      -- 导入时间
    lastReadAt INTEGER,               -- 最后阅读时间
    readProgress INTEGER DEFAULT 0,   -- 阅读进度（页码）
    totalPages INTEGER,
    isFavorite BOOLEAN DEFAULT 0,
    isArchived BOOLEAN DEFAULT 0,
    notes TEXT                        -- 用户笔记
);

-- 标签表（主题、关键词等）
CREATE TABLE tags (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    type TEXT NOT NULL,               -- 'topic', 'keyword', 'custom'
    color TEXT,                       -- 标签颜色 #RRGGBB
    createdAt INTEGER NOT NULL
);

-- 论文-标签关联表
CREATE TABLE paper_tags (
    paperId TEXT NOT NULL,
    tagId TEXT NOT NULL,
    PRIMARY KEY (paperId, tagId),
    FOREIGN KEY (paperId) REFERENCES papers(id) ON DELETE CASCADE,
    FOREIGN KEY (tagId) REFERENCES tags(id) ON DELETE CASCADE
);

-- 收藏/高亮内容
CREATE TABLE highlights (
    id TEXT PRIMARY KEY,
    paperId TEXT NOT NULL,
    pageNumber INTEGER NOT NULL,
    selectedText TEXT,
    note TEXT,                        -- 用户注释
    color TEXT DEFAULT '#FFD700',     -- 高亮颜色
    rect TEXT,                        -- 矩形区域（JSON: {x, y, width, height}）
    createdAt INTEGER NOT NULL,
    FOREIGN KEY (paperId) REFERENCES papers(id) ON DELETE CASCADE
);

-- 对话历史表
CREATE TABLE conversations (
    id TEXT PRIMARY KEY,
    paperId TEXT,
    topic TEXT NOT NULL,              -- 对话主题
    createdAt INTEGER NOT NULL,
    updatedAt INTEGER NOT NULL,
    messageCount INTEGER DEFAULT 0,
    contextSummary TEXT,              -- 上下文摘要（用于压缩）
    FOREIGN KEY (paperId) REFERENCES papers(id) ON DELETE SET NULL
);

CREATE TABLE messages (
    id TEXT PRIMARY KEY,
    conversationId TEXT NOT NULL,
    role TEXT NOT NULL,               -- 'user' | 'assistant'
    content TEXT NOT NULL,
    contentType TEXT DEFAULT 'text',  -- 'text' | 'image' | 'mixed'
    metadata TEXT,                    -- JSON: {pageNumber, selectedText, etc.}
    timestamp INTEGER NOT NULL,
    FOREIGN KEY (conversationId) REFERENCES conversations(id) ON DELETE CASCADE
);
```

## 10. 技术难点与解决方案

### 10.1 PDF 文本提取精度

- **方案**: 使用 pdfjs-dist + 自定义文本层
- **备选**: 对扫描版 PDF 使用 OCR (Tesseract.js)

### 10.2 长对话上下文管理

- **方案**: 按话题分会话 + 用户主动管理
- **优化**: 提供 "总结对话" 按钮生成摘要

### 10.3 大量论文性能

- **方案**: 数据库索引 + 虚拟滚动
- **分页**: 图书馆页面分页加载

### 10.4 AI 接口兼容性

- **方案**: 统一使用 OpenAI Chat Completions 兼容格式，通过配置 baseUrl 适配不同平台
- **流式输出**: 使用 SSE (Server-Sent Events) 实现流式响应
- **错误处理**: 包装调用，提供重试机制，网络错误时显示友好提示
- **多模态**: 支持 vision 模型的图片输入（截图提问），不支持时降级为 OCR + 文本
